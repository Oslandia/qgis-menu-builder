Créer ses propres menus dans QGIS
et ajouter des raccourcis vers ses couches préférées



Un plugin pour ajouter assez simplement un ou plusieurs nouveau(x) menu(s) avec :

  • des sous-menus pour organiser ses raccourcis,
  • des raccourcis vers les couches de son choix (raster, vecto, WMS ou WFS),
  • des raccourcis vers des fichiers projets Qgis,
  • des liens vers des sites internet (s'ouvrent dans le navigateur par défaut),
  • des raccourcis vers d'autres fichiers (s'ouvrent avec l'application liée à leur extension),
  • des « programmes » (fichiers python) pour ajouter des fonctions à Qgis,
  • des séparateurs (lignes horizontales dans un menu)


Il permet aussi de forcer l'affichage ou bien de masquer des barres d'outils.


        Démarrage :        

Installer l'extension depuis le dépôt MEDDTL.
Au premier démarrage de l'extension, elle va créer un fichier Menus.xml dans le dossier utilisateur de Qgis (%USERPROFILE%\.qgis2 ou C:\Program Files\QGIS\profil) avec un menu Raccourcis et quelques exemples de couches (dont les raccourcis vers les WMS de GéoRef).
(par ex. l'utilisateur TOTO aura le fichier : C:\Users\TOTO\.qgis2\Menus.xml )
Vous pourrez personnaliser vos menus en modifiant ce fichier (voir le § suivant).
Si vous déplacez ce fichier, il faudra indiquer à l'extension où vous l'avez placé : dans Qgis, menu Extension  puis  Créer ses propres menus  puis  Définir l'emplacement du fichier qui décrit les menus



        Détails du fichier Menus.xml        

Voir l’exemple Menus.xml.modele dans le dossier de l'extension. Un programme comme Notepad++ est recommandé pour éditer ce fichier qui respecte la norme XML.
La balise « racine » est <menus>. Elle contient une branche <options> et une ou plusieurs branches <menu>.
Les commentaires se placent entre la marque  <!--  et  la marque  -->
Les lignes qui se trouvent dans une section commentaire ne sont pas traitées par l'extension.


Des paramètres optionnels sous la balise <options> :

L'option <MenusCommuns> permet d'avoir un autre fichier xml commun partagé (par exemple sur le réseau). Par exemple :
<MenusCommuns>R:\GEOBASE\MenusQGIS.xml</MenusCommuns>
Il faut indiquer le chemin complet vers ce fichier (ex.: \\Serveur\SIG\MenusCommuns.xml)
Toutes les options ainsi que les menus décrits dans ce fichier commun seront traités en premier, ensuite les options et menus du Menus.xml seront traités à leur tour.

L'option <AjoutMenu> permet de choisir où vont apparaître les menus décrits ci-dessous. Par défaut, ils sont placés dans la barre principale des menus de Qgis (à droite de "Aide").
Cette option ajoute les possibilités suivantes:
    Principal  (paramètre par défaut: ajoute les menus à la barre des menus)
    Extension  (ajoute les menus dans le menu Extension)
    Couche     (ajoute les menus dans le menu Couche)
    Database  (ajoute les menus dans le menu Base de donnée)
Par exemple:
    <AjoutMenu>Couche</AjoutMenu>


Les options <barre permettent de forcer l'affichage ou bien de masquer des barres d'outils.
Voici les noms des différentes barres : BarreAide, BarreAttribut, BarreCouches, BarreEtiquette, BarreFichier, BarreGRASS, BarreNavig, BarreNum, BarreNumAvancée, BarrePlugin, BarreRaster, BarreVecteur, BarreBase et BarreInternet
Par exemple:
    <barre nom="BarreRaster" afficher="N" />
    pour cacher une barre : afficher="N"
    pour afficher une barre : afficher="O"


Chaque menu est décrit dans une balise <menu>

exemple : <menu nom="Mes raccourcis">
Il peut contenir :

  • des sous-menus (même fonctionnement qu'un menu de base) : <menu ... 
  • des actions (une entrée du menu qui active différents types d'actions) : <action
  • des séparateurs (ligne horizontale dans le menu pour distinguer visuellement des groupes d'actions) : <séparateur /> 

 

Remarque importante : les caractères  & et  <
ils doivent impérativement être remplacés respectivement par : &amp;  et  &lt; 
Exemple, pour un fichier Parcelles&Bati.qgs :
<action nom="Cadastre" type="projet">Parcelles&amp;Bati.qgs</action>

Attention : dans un menu, le & a une utilisation spéciale. Utilisé seul dans le nom de l'action, il ne va pas être affiché mais il va créer un raccourci clavier vers la lettre qui le suit. &Cadastre affichera Cadastre (et affectera la lettre C comme raccourci vers cette action). Pour afficher un & dans un menu, il faut écrire &&. Donc pour afficher l'action « Parcelles & Bâti », il faudra écrire : nom="Parcelles &amp;&amp; Bâti"  dans Menus.xml !


        Les Actions :        

Exemple :
 <action nom="Pentes" type="vecteur">U:\BDALTI\PENTES.TAB</action>
L'attribut nom indique le libellé qui sera affiché dans le menu.
L'attribut type indique comment interpréter et traiter le texte de cette action.

Remarque sur les fichiers :

  • ils peuvent être placés n'importe où (sur le PC ou en réseau)
  • ces actions sont ajoutées au menu uniquement si le fichier existe réellement
  • indiquer un chemin absolu ou bien relatif par rapport au fichier xml qui les appelle (soit le Menus.xml, soit un MenusCommuns.xml partagé)

Ex. si on a \\sbl1\SIG\MenusCommuns.xml pour ouvrir \\sbl1\SIG\Cartes\Risques.qgs  on peut écrire:
<action nom="Risques" type="projet">\\sbl1\SIG\Cartes\Risques.qgs</action>
mais on peut aussi écrire :
<action nom="Risques" type="projet">Cartes\Risques.qgs</action>

Ajouter une couche vectorielle : type="vecteur"

L'action doit être un chemin d’accès à un fichier vectoriel reconnu par Qgis.
Exemple :
 <action nom="Région" type="vecteur">R:\BDCARTO\REGION.TAB</action>
Remarque : utilisation de fichier de style .qml :
Si on place un fichier .qml avec le même nom qu'une couche vecteur et dans le même dossier que le xml qui l'appelle (soit le Menus.xml, soit un MenusCommuns.xml partagé), ce style sera appliqué à l'ouverture de la couche.
Exemple :
Si on place un REGION.qml dans le même dossier que le xml où on trouve l'action ci-dessus, le menu appliquera ce style si on clique sur l'action "Région"


Ajouter une couche raster: type="raster"

L'action doit être un chemin d’accès à un fichier raster reconnu par Qgis.
Remarque : le raster sera placé en dernière position dans la légende (si un groupe est "actif" le raster ira à la fin de ce groupe)
Exemple :
 <action nom="BDORTHO" type="raster">R:\BDORTHO\ORTHO.ecw</action>
Remarque : utilisation de fichier de style .qml :
Si on place un fichier .qml avec le même nom qu'une couche raster et dans le même dossier que le xml qui l'appelle (soit le Menus.xml, soit un MenusCommuns.xml partagé), ce style sera appliqué à l'ouverture de la couche.
Exemple :
Si on place un ORTHO.qml dans le même dossier que le xml où on trouve l'action ci-dessus, le menu appliquera ce style si on clique sur l'action "BDORTHO"


Ajouter un fichier de Définition de Couche (.qlr): type="qlr"

L'action doit être un chemin d’accès à un fichier .qlr (accès direct à une couche avec des styles).
Exemple :
 <action nom="Forêts (postgis)" type="qlr">C:\DONNEES\forets.qlr</action>


Ajouter une couche Spatialite: type="spatialite"

L'action doit être le texte de connexion à la base de données.
Exemples :
 <action nom="Communes" type="spatialite"> dbname="R:\bdcarto\admin.sqlite" table="communes" (the_geom) sql=SUPERFICIE>5000</action>
 <action nom="Parcelles de Maville" type="spatialite">dbname="R:\cadastre\bdp.sqlite" table="parcelles" (Geometry) sql=NOM_COMM='Maville'</action>
 <action nom="Requête Spatialite" type="spatialite"> dbname="R:\cadastre\bdp.sqlite" key='id' table="(SELECT rowid AS id, code_com || com_abs || section AS idsec, ST_Union(geom) as geom FROM divcad GROUP BY idsec)" (geom)</action>


Ajouter une couche PostreSQL/PostGIS: type="postgres"

L'action doit être le texte de connexion à la base de données.
Exemples :
 <action nom="Bâtiments" type="postgres"> host=monserveur port=5432 user='toto' password='motdepasse' dbname='mabase' table="monshema"."batiments" (the_geom)</action>
 <action nom="Parcelles de Maville" type="postgres"> host=monserveur port=5432 user='toto' password='motdepasse' dbname='mabase' table="monshema"."parcelles" (geom_s) sql=commune='Maville'</action>


Ajouter une couche WMS : type="wms"

L'action doit comporter les 2 sous-balises <url> et <couche> ; par exemple :
 <url>http://georef.application.i2/cartes/mapserv</url>
 <couche>noms_regions</couche>

2 autres sous-balises sont optionnelles :

  • <format> (valeur par défaut : image/png )
  • <scr> (valeur par défaut : EPSG:2154 ) 


Exemple d'une couche en WGS84 (EPSG:4326) :
  <action nom="Limites Administratives" type="wms">
    <url>http://georef.application.i2/cartes/mapserv </url>
    <couche>limites_administratives</couche>
    <scr>EPSG:4326</scr>
 </action>
 

Ajouter une couche WFS : type="wfs"

L'action doit comporter les 2 sous-balises <url> et <couche> ; par exemple :
 <url>http://serveur.web/wfs</url>
 <couche>communes</couche>

3 autres sous-balises sont optionnelles :

  • <version> ( version du WFS ; valeur par défaut : 1.0.0 )
  • <scr> ( projection : valeur par défaut : EPSG:2154 )
  • <cache> oui/non ( indique si Qgis télécharge toute la couche dés l'ouverture ; valeur par défaut : non )


Exemple d'une couche en Lambert II (EPSG:27582) :
  <action nom="Limite des marées (WFS)" type="wfs">
  <url>http://geolittoral.application.equipement.gouv.fr/wfs/metropole</url>
  <couche>Limites_marees</couche>
  <scr>EPSG:27582</scr>
  <cache>oui</cache>
 </action>


Ouvrir un projet .qgs : type="projet"

L'action doit être un chemin d’accès à un fichier projet Qgis.
Cela ouvre une boite de dialogue qui propose: "Tout fermer et ouvrir ce projet" ou "Intégrer les couches de ce projet"
Exemple :
 <action nom="Risques" type="projet">R:\SIG\Risques.qgs</action>


Ajouter les couches d'un projet .qgs : type="multi"

L'action doit être un chemin d’accès à un fichier projet Qgis.
Par exemple, ajouter au travail en cours les couches du projet Carte_Risques :
 <action nom="Risques" type="multi">Carte_Risques.qgs</action>


Accéder à un site web : type="web"

L'action doit être un lien internet et commencer par le texte :  http
La page va s'ouvrir dans le navigateur par défaut.
Remarque : ce type d'action ne fonctionne que sur les systèmes Windows.
Exemple :
 <action nom="GéoInfo" type="web">http://geoinfo.metier.i2</action>
 

Ouvrir un autre type de fichier ou un répertoire type="fichier"

L'action doit être le chemin d’accès à un fichier ou à un dossier. Un fichier sera ouvert avec l'application définie par défaut dans Windows pour ce type de fichier. Un dossier sera ouvert dans l'explorateur Windows.

Remarque : ce type d'action ne fonctionne que sur les systèmes Windows.
Exemples :
  <action nom="BDTOPO" type="fichier">\\LINDA\GB_REF\BDTOPO</action>
  <action nom="Docs urba" type="fichier">D:\Urba\Etat.xls</action>
 

Exécuter du code python type="python"

L'action doit être le chemin d’accès à un fichier python. Il doit avoir une classe action() avec
   def __init__(self, iface)
Le paramètre iface est l'interface Qgis, ce qui permet d'interagir avec l'application.

   exemple : cacher toutes les barres d'outils d'un coup :

from PyQt4.QtGui import *
class action():
   def __init__(self, iface):
      barres = iface.mainWindow().findChildren(QToolBar)
      for b in barres:
         b.hide()



exemple ToutFermer.py : fermer toutes les couches d'un coup :

from qgis.core import QgsMapLayerRegistry
class action():
  def __init__(self, iface):
    for L in iface.legendInterface().layers():
      QgsMapLayerRegistry.instance().removeMapLayer(L.getLayerID())


exemple :
  <action nom="Fermer tout" type="python">ToutFermer.py</action>
 

Un exemple de fichier Menus.xml :

<?xml version='1.0' encoding='windows-1252'?>
<menus>
  <options>
    <barre nom="BarreEtiquette" afficher="O" />
  </options>

  <menu nom="Mes raccourcis">
    <action nom="Fermer toutes les couches" type="python"> ToutFermer.py </action>
    <séparateur />
    <action nom="Communes" type="vecteur"> U:\BDCARTO\COMMUNE.TAB </action>
    <action nom="Dossier BDCARTO" type="fichier">U:\BDCARTO</action>
    <action nom="Risques" type="projet">Projets\Carte_Risques.qgs</action>

    <menu nom="Couches WMS">
      <action nom="Réseau Routier National" type="wms">
        <url>http://georef.application.i2/cartes/mapserv</url>
        <couche>reseau_routier_national</couche>
      </action>
    </menu>
  </menu>
</menus>
